# テクニカルワークフロー{#about-technical-workflows}

## テクニカルワークフローについて {#overview}

この節で詳しく説明するワークフローは、別の Adobe Campaign ビルトインパッケージと共にインストールされます。これらのパッケージおよび関連テクニカルワークフローは、ライセンス契約に応じて異なります。組み込みパッケージの詳細については、[このセクション](../../installation/using/installing-campaign-standard-packages.md)を参照してください。

デフォルトでは、テクニカルワークフローは次のノードのサブフォルダーで使用できます。**[!UICONTROL 管理]** > **[!UICONTROL 実稼動]** > **[!UICONTROL テクニカルワークフロー]**。

>[!NOTE]
>
>Message Center モジュールに関連するテクニカルワークフローは、デフォルトの状態で&#x200B;**[!UICONTROL 管理]**／**[!UICONTROL プロダクション]**／**[!UICONTROL Message Center]**／**[!UICONTROL テクニカルワークフロー]**&#x200B;ノードに用意されています。

テクニカルワークフローの監視方法について詳しくは、[該当する節](../../workflow/using/monitoring-technical-workflows.md)を参照してください。

| **予測** （予測） | 配信 | 暫定カレンダー（暫定ログを作成）に保存された配信を分析します。デフォルトで、毎日午前 1 時にトリガーされます。 |

| **変換済みの連絡先のID** (webAnalyticsFindConverted) | Web 分析コネクタ | リマーケティングキャンペーンの後で、購入を完了したサイトの訪問者をインデックス化します。このワークフローで回復されたデータは、再マーケティングの効率性レポートでアクセスできます（このページを参照）。 |
| **Adobe Experience Cloudからのオーディエンスの読み込み** (importSharedAudience) | Adobe Experience Cloudとの統合 | このワークフローでは、別の Adobe Experience Cloud ソリューションのオーディエンスおよびセグメントを Adobe Campaign にインポートできます。 |
